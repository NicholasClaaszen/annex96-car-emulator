CompatibilityRaspberry Pi series 2, 3 and 4DC power input is reverse-polarity safe. However; since automotive applications utilize EARTH as ground reference,reverse polarity will result in a short circuit of the external power supply.Compatibility of the HAT is limited to Raspberry Pi versions equipped with the standard 40 pin GPIO header. In some cases,the shutdown function must be wired externally.The total power consumption of the system hads to be limited to 10W. This should be taken into account when usingexternal (USB) equipment and or a Raspberry Pi 4.Functional descriptionPLC-HAT can be used in conjunction with an EVSE to extend the functionality with PLC based communication to an EV .The PLC-HAT provides all functions needed to interact with an IEC61851-1 compatible AC/DC charging station in mode 3 and 4. the HAT can beset-up as an EVCC or SECC. In EVCC mode the HAT can sample the CP signals. With the power-Down functionality, The raspberry Pi can beshut down and woken by the microcontroller when a change on CP voltage is detected. PLC-HAT could also be used to retrofit basic 15118(Plug&Charge) support into an EV for pilot purposes.PowerPLC-HAT can be used to back-power the raspberry pi by connecting a DC supply to J2. Acceptable voltage levels are between 9 and 30 voltsmaking the application suitable for retrofit in charging stations and EV’s.HAT RecognitionPLC-HAT is equipped with an ID-EEPROM in conformance with the Raspberry Pi HAT specification. During boot, the Raspberry reads thecontents of the EEPROM and loads the appropriate drivers and GPIO settings. Powering from both external and microUSBIn case the system is powered via both the Raspberry Pi micro-USB connector and J2 from the PLC-HAT; The ideal diode constructed with Q3will protect the circuits in conformity with the Raspberry Pi HAT specification.MicrocontrollerPLC-HAT is equipped with an Atmel ATMEGA328p microcontroller. The microcontroller is in control of sampling of CP signals, switching theMode3 state between state A, B and C, and powering up/down the Raspberry Pi for energy management. The Microcontroller has a serial UARTconnection to the Raspberry Pi through Serial0.Input voltage limitationsFor circuit protection, the CP is protected by TVS device D1.200Figure 4. Input protectionCP is protected by Bi-directional TVS diode with stand-off voltage of 13V . The breakdown voltage of this device is 15.90V max. The maximuminput voltage on CP-in without damaging the microcontroller is around ±16 V olts, making the circuit effectively protected. Putting a higher voltagethan 15.9 on the input will start D1 to conduct, increasing the current and limiting the voltage on the input to max 15.9V .Control Pilot detectionThe Control Pilot signal is fed through R4 and scaled by voltage divider (R8/R9) to a voltage level between approx 0.7 and 2.7 volt (for ±12Vinput). This signal, further called CP_ANALOG, is fed to PIN AD6 of the microcontroller for analog sampling.200Figure 5. CP InputThe CP_ANALOG signal is fed to U6, a logic buffer with Schmitt-trigger input, shaping the CP input to a clean square wave, sampled on PIN B0(arduino pin 8) of the Atmega for duty cycle calculation.EV EmulationPLC-HAT can optionally be equipped with a switching circuitry for EV emulation. Via microcontroller pin D5 and D6, the state of the CP line canbe set to state B and C. Where B is an indication that an EV is connected, and C that the EV is requesting energy.
