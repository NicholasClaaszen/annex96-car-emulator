IntroductionThis document describes the Application Controller API implemented on the PLC_HAT.Serial communicationThe Application Controller is connected to tty0 (dev/serial0) of the Raspberry via the GPIO header. Communication speed is set at 115200 baudwith 8 bits per character with 1 stop bit (115200 8N1).Command and argument are separated by a colon (:). The Application Controller will parse an incoming string once a semi-colon (;) is received.When an incoming string doesn’t comply to a command; the Application Controller responds by sending "CMD_ERR;".All messages from the Application Controller are terminated by a semi-colon. NOTE: line break, carriage return and space characters in incomingstrings are ignored.Boot and communication initiationAfter power-on the Application controller will repeatedly send 'wdt=timedout' and 'wdt=resettriggered' at a rate of 2 times the watchdog interval,until the watchdog timer is reset or other communication has taken place. In this state the CP line is kept in state A.WatchdogThe Application Controller implements a watchdog function. In the event the watchdog is triggered; the CP state is reset to state A. Behavior willbe as described in chapter Boot and communication initiation. The default interval is 5 seconds. this can be altered via set_wdt_intervalThe Raspberry Pi can 'pat' the watchdog by responding to the 'wdt=timedout' message from the Application Controller. see alive ping for moreinformation.By default the watchdog is enabled. It can be disabled by setting watchdog to disabled.The internal watchdog timer of the Application Controller is also reset on every successful message interchange.During communication init and in sleep mode; the watchdog function is disabled.Custom watchdog time-out intervals will revert to default after an Application Controller reset.ShutdownBy sending 'shutdown:<xxx>'; the raspberry can instruct the application controller to go into power safe mode. The Application Controller willdisable the Raspberry Pi voltage regulator by pulling PEN low.The Application controller will periodically sample the voltage on the Control Pilot line. When a valid state is detected (state A or F) the Raspberryis energized and the Application Controller reboots; resulting in periodically overflow of the watchdog timer.In low power mode, the Application Controller monitors the CP voltage. When a state change occurs, the Application Controller will re-activatethe Raspberry.EV[SE] Raspberry Pi HATﬁle:///Users/klaas/Downloads/microcontroller.adoc

Getting CP statusRaspberry can request the status of the Control Pilot by the getstate command.Values that can be requested are CP state, voltages and duty-cycle.Setting CP statusRaspberry can set the status of the Control Pilot by the setstate command.The CP can be set to A, B, and C in conformance with the IEC61581-1 Mode 3 specification.Automatic report of state changesThe Application controller will autonomously inform the Raspberry of detected changes in the Control Pilot state concerning PWM duty-cycle andstate.Changes in duty-cycle are reported via pwmchanged. Changes in Mode 3 state are reported via statechanged.Automatic report of PWM changes is enabled on boot. This behavior is controlled by sending report_pwm_changes to disable. Automatic report ofControl Pilot state changes is enabled on boot. This behavior is controlled by sending report_cp_changes to disable.Only PWM changes of 1% and more are automatically reported.Please note that due to no response is expected, this will not reset the watchdog timer.Application controller APIThis chapter contains te message definition of the Application Controller API.From Raspberry to Application ControllerMessageDescriptioncp_filterSet CP detection filter valuegetstateget current state of Control Pilotreport_cp_changesEnable / disable automatic report of state changes in CPreport_pwm_changesEnable / disable automatic report of PWM changes in CPsetstateset Control Pilot to desired statesetppSet PP on/offset_wdt_intervalSet the watchdog timer intervalshutdownSet application to sleepEV[SE] Raspberry Pi HATﬁle:///Users/klaas/Downloads/microcontroller.adoc
